📊 Adult Census Population ClusteringAn Unsupervised Machine Learning Project🌟 Project OverviewThis project explores the "Adult" Census dataset to find hidden patterns in how different groups of people are segmented based on their demographics and income. Using R, I transformed raw data into meaningful groups (clusters) to understand the socio-economic landscape of the population. 🛠️ Technology StackI used the R Programming Language along with several specialized "libraries" (toolkits) to handle the data:Tidyverse: The Swiss Army knife for data cleaning and visualization.Cluster & Factoextra: Specialized tools for performing the actual clustering math and making the results look visual.DBSCAN: A tool used specifically for finding "dense" groups of data and ignoring "noise" (outliers).Plotly: Used to create interactive graphs that you can zoom into.🛣️ The Data Journey (Step-by-Step)1. Data Pre-processing (Cleaning the Mess)Before the AI can learn, the data must be "cleaned."Filtering: I removed columns like capital_gain and fnlwgt because they were mostly empty or confusing, which would have distracted the model. Recoding: I simplified complex categories. For example, instead of having five different types of "Government" jobs, I grouped them all into one category called gov. Scaling: Imagine comparing a person's Age (0-100) to their Income (0-100,000). The computer might think Income is "more important" just because the numbers are bigger. I "scaled" the data so every variable is treated equally. 2. Finding the "Magic Number" of GroupsHow many clusters do we need? I used three mathematical tests:The Elbow Method: Looking for the point where adding more clusters doesn't provide much more "value." Silhouette Method: Measuring how well each person fits into their assigned group versus the next closest group. 3. Applying the AlgorithmsI used three different ways to group the data:K-Means: The most popular method. It picks a center point for each group and tries to keep everyone close to it. Hierarchical Clustering: This builds a "family tree" (Dendrogram) for the data, showing how groups branch off from one another. DBSCAN: This method looks for "crowds." If a data point is in a crowded area, it's part of a cluster. If it's all alone, it's labeled as "noise." 📈 Key InsightsHigh Earners: One clear group consisted of married, white males with Bachelor’s or Master’s degrees working in the private sector. Students & Entry-Level: Another group was younger (ages 25-35) with lower income, often single, and still pursuing college education. Unique Segments: Hierarchical clustering successfully identified specific subgroups, such as widowed individuals with varied education levels, which other methods missed.
